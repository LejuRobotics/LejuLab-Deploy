GlobalRobot

class GlobalRobot

全局机器人单例管理器

提供统一的机器人创建和管理接口，确保全局只有一个机器人实例。

// 从环境变量初始化
GlobalRobot::init_env(RobotVersion::from_env());
auto& robot = GlobalRobot::getInstance();
robot.publishRobotCmd(cmd);

// 使用预定义版本常量
GlobalRobot::init_env(RobotVersions::KUAVO5_BASE);

Public Static Functions

static bool init_env(const RobotVersion &version)

初始化机器人运行环境（只能调用一次）

参数:

version – 机器人版本信息，参见 RobotVersion

抛出:

std::runtime_error – 重复初始化时抛出

返回:

初始化是否成功

static RobotBaseAPI &getInstance()

获取已初始化的机器人实例

必须在调用 init_env() 之后使用，否则将抛出异常。

// 从环境变量中获取机器人版本信息
auto robot_version = RobotVersion::from_env();
if (GlobalRobot::init_env(robot_version)) {
    auto& robot = GlobalRobot::getInstance();
    robot.publishRobotCmd(cmd);
}

// 使用预定义的版本常量 - Kuavo5
if (GlobalRobot::init_env(RobotVersions::KUAVO5_BASE)) {
    auto& robot = GlobalRobot::getInstance();
}
抛出:

std::runtime_error – 未初始化时抛出

返回:

机器人实例引用

static bool is_initialized()

检查机器人环境是否已初始化

返回:

已初始化返回 true

TopicLogger

class TopicLogger

基于 DDS 的日志发布器

用于动态发布调试数据到 DDS Topics，支持向量和标量数据类型。 自动管理 Topic 生命周期，线程安全。

// 创建 Logger (使用默认 domain id = 0)
auto logger = leju::TopicLogger::create();

// 或指定 domain id
auto logger = leju::TopicLogger::create(1);

// 发布数据
std::vector<double> joint_pos = {0.1, 0.2, 0.3};
logger->publishVector("/controller/joint_pos", joint_pos);
logger->publishValue("/monitor/loop_time", 0.5);
